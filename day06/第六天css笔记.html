第六天css笔记

想让两个块级元素在一行显示，转换为行内块存在的一些问题。
问题一：元素之间会有一个缝隙，产生的原因就是（换行引起的）

解决的办法：
1、消除换行符，让代码在一行显示
2、给它的父级写一个font-size:0;如果有元素里面有文字，可以单独对它设置文字大小。

问题二：如果元素里面有文字，默认的对齐方式是基线对齐，会造成一定的问题。

解决的办法：vertical-align：top 或者vertical-align：middle;等。

浮动

使用场景：想让块级元素在一行显示，可以用浮动。
浮动的几个值float:

  • 左浮动：float:left；

  • 右浮动：float:right;

  • float:none(默认值)；

浮动的特点

  • 1、脱离文档流

  • 2、浮动的方向是相对父级来说的。父亲是它的参照物

  • 3、一般来说宽度是可继承的，一旦写了浮动，这个宽度就不能继承父亲的宽度，而是由自身内容大小决定的。

  • 4、一旦设置浮动，行内元素设置宽高也能起作用。

浮动造成盒子塌陷解决方法：

  • 1、给父级盒子加高度

  • 2、给浮动元素的父级的末尾添加一个空div，给它设置clear：both；

    • clear：left；清除左浮动

    • clear：right；清除右浮动

    • clear：both；清除左右浮动（通常用这个）

  • 3、谁浮动，就给谁的父级的类名中添加clearfix。这种方法和第二种的本质一样，只是这种让你写起来更方便。（用的最多）

  • 4、给浮动元素的父级添加overflow：hidden；

.clearfix:after{

        display:block;

        content:"";

        height:0;

        font-size:0;

        visibility: hidden;

        clear:both;

    }

    .clearfix{

        *zoom:1;

    }

浮动的本质：最初是为了实现图文混排。场景，一张图片和一些文字，当我们给图片设置浮动的时候，文字就会环绕图片进行排版。

伪类

需要注意：content即使里面你不需要写内容，咱们也要把这个content:” “;也要写上。
使用场景，如果不想在结构中增加元素了，就可以用这种方式，比较常见的就是列表前面的icon;

<style>

       p::before{

           display:inline-block;

           content:"大家都很棒！";

           color:red;

       } 

       p::after{

           display: inline-block;

           content:"大家成功毕业";

           color:green;

       }

</style>

overflow 几个值：

  • hidden；超出隐藏

  • auto；当内容大于盒子的宽高的时候，会自动加滚动条

  • scroll; 不管内容多少，始终加滚动条

background几个常用的属性

  • background-color:背景颜色

  • background-image:url(“”);背景图片

  • background-position 背景位置

    • 水平：left、center、right

    • 垂直：top、center、bottom

    • 用的比较多的center、center

  • background-attachment:fixed/scroll;

  • background-repeat:

    • x 轴：repeat-x；

    • y轴：repeat-y;

    • no-repeat 不平铺

  • background-size:

    • 可以跟具体的像素值，例如background-size:50px 50px;让图片的宽是50px,高也是50px;

    • 百分比background-size:100% 100%;

    • cover;为了把图片铺满整个容器，有可能会让图片的一部分内容不能显示完全。

    • contain：以最合适的方式进行缩放，只要一边能够铺满宽或者高，就停止放大。

绝对路径和相对路径

  • 绝对路径：不管是内部还是外部的链接，都能访问到，例如带域名的线上的网址或者是电脑磁盘中的路径。

  • 相对路径：如果是同级，直接写名字，如果是下一级，用”/ “;如果是上一级用“../”,如果是上两级就是“../../”,依次类推。
